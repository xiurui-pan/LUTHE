# WoP-PBS (Programmable Bootstrapping Without Padding) 实现

## 概述

WoP-PBS是TFHE中的一个重要变体，它实现了无填充的可编程自举算法。**新的简化设计**基于一个关键洞察：**WoP-PBS本质上是使用不同LUT的多个标准PBS操作的序列**。通过这种认识，我们能够最大化复用现有的`pe_pbs.sv`模块，大幅简化实现复杂度。

## 核心算法映射

### WoP-PBS三阶段算法

根据您提供的算法流程，WoP-PBS包含三个阶段：

#### 第一阶段：比特提取 (Bit Extraction)
- **目标**: 将高精度消息（如16比特）拆分成多个低精度块，提取每个比特
- **实现**: 使用多次标准PBS，每次使用专门的比特提取LUT
- **输出**: 每个比特单独加密在LWE密文中

#### 第二阶段：电路自举 (Circuit Bootstrapping)  
- **目标**: 将LWE比特密文转换为GGSW密文格式
- **实现**: 使用标准PBS，配合电路自举专用LUT
- **输出**: GGSW密文，作为下一阶段的选择器

#### 第三阶段：垂直封装 (Vertical Packing)
- **目标**: 使用GGSW比特密文评估大型LUT函数
- **实现**: 使用标准PBS，配合最终的大型评估LUT
- **输出**: 最终计算结果

### 关键设计洞察

**WoP-PBS = 多个PBS + 不同的LUT**

这个洞察使我们能够：
1. **完全复用** `pe_pbs.sv` 模块
2. **简化架构** - 无需重新实现复杂的NTT引擎或外部积逻辑
3. **降低风险** - 使用已经验证的PBS实现
4. **提高可维护性** - 减少代码重复

## 新的简化架构

### 主要模块

#### 1. `wop_pbs_kernel_simplified.sv` - 简化主内核
- **功能**: WoP-PBS的顶层控制器
- **设计**: 包含指令管理器 + 标准PBS实例
- **复杂度**: 大幅简化，主要是连线和控制逻辑
- **状态**: ✅ 实现完成

#### 2. `wop_pbs_instruction_manager.sv` - 指令管理器
- **功能**: 将WoP-PBS指令转换为PBS指令序列
- **核心逻辑**: 三阶段状态机，生成适当的PBS指令
- **职责**: 
  - 解码WoP-PBS指令
  - 管理中间结果地址
  - 生成每个阶段的PBS指令
- **状态**: ✅ 实现完成

#### 3. `pe_pbs.sv` - 复用的标准PBS
- **功能**: 执行所有实际的计算工作
- **修改**: **无任何修改** - 完全复用现有实现
- **优势**: 
  - 已验证的实现
  - 完整的NTT引擎支持
  - 所有优化都可直接受益

### 移除的复杂模块

#### ❌ 移除的模块

- `sample_extractor.sv` - 不再需要专门的样本提取器

## 与标准PBS的对比

| 特性 | 标准PBS | WoP-PBS |
|------|---------|---------|
| **输入** | 单个LWE密文 | 高精度LWE密文 |
| **LUT** | 单个查找表 | 多个专用LUT |
| **操作次数** | 1次PBS | 多次PBS (bit_width + 1) |
| **输出** | 单个LWE密文 | 大型LUT评估结果 |
| **硬件复用** | - | 100%复用PBS |

## 性能分析

### 1. **延迟**
- **比特提取阶段**: bit_width × PBS_latency
- **电路自举阶段**: bit_width × PBS_latency  
- **垂直封装阶段**: 1 × PBS_latency
- **总延迟**: (2 × bit_width + 1) × PBS_latency

### 2. **吞吐量**
- 受限于PBS的吞吐量
- 可通过流水线优化提升
- 支持多个WoP-PBS实例并行

### 3. **资源使用**
- **LUT**: 与PBS相同
- **BRAM**: 与PBS相同 + 少量地址管理
- **DSP**: 与PBS相同
- **额外开销**: <5%

## 技术细节

### 指令流程
```
WoP-PBS指令 → 指令管理器 → PBS指令序列 → PBS执行 → 结果
```

### 地址空间管理
- **输入区域**: 原始LWE密文
- **临时区域**: 比特提取结果 + 电路自举结果
- **输出区域**: 最终结果

### LUT组织
- **比特提取LUT**: 每个比特位一个LUT
- **电路自举LUT**: 每个比特位一个LUT  
- **垂直封装LUT**: 单个大型LUT (2^bit_width 条目)

## 总结

新的简化WoP-PBS设计通过**复用而非重造**的策略，大幅降低了实现复杂度，同时保持了高性能和可靠性。这种方法不仅减少了开发工作量，还确保了与现有系统的完美兼容性。

关键成功因素：
1. **正确的抽象层次** - 在指令级别而非硬件级别处理WoP-PBS逻辑
2. **最大化复用** - 100%利用现有PBS实现
3. **清晰的模块分离** - 指令管理与计算执行分离
4. **渐进式开发** - 可分阶段实现和验证
