# WoP-PBS (Programmable Bootstrapping Without Padding) 实现

## 概述

WoP-PBS是TFHE中的一个重要变体，它实现了无填充的可编程自举算法。本实现基于Zama HPU的现有架构，最大化复用了标准PBS流水线的核心组件。

## 核心算法映射

### C++算法到RTL的映射

基于`circuit_bootstrapping.cpp`中的`circuitBootstrapWoKS`函数，算法流程如下：

1. **测试向量加载**: 从HBM加载预计算的测试向量到累加器BRAM
2. **输入LWE加载**: 从RegFile加载输入LWE样本的'a'部分
3. **主循环 - 盲旋转**:
   - `acc2 = (X^aibar - 1) * acc1` (多项式旋转和减法)
   - `acc1 = BKi * acc2` (外部积计算)
   - `acc += acc1` (累加)
4. **样本提取**: 从最终累加器中提取LWE样本

## 模块架构

### 主要模块

#### 1. `wop_pbs_kernel.sv` - 主内核模块
- **功能**: 实现完整的WoP-PBS算法流程
- **状态机**: 10个状态，直接映射到C++算法逻辑
- **接口**: 与共享NTT引擎、BSK管理器、RegFile和AXI接口交互
- **状态**: ✅ 框架完成，需要实现具体逻辑

#### 2. `poly_rotator_sub.sv` - 多项式旋转器
- **功能**: 实现`(X^aibar - 1) * acc1`操作
- **算法**: 多项式旋转 + 减法
- **特点**: 处理X^N = -1的符号翻转
- **状态**: ✅ 实现完成

#### 3. `sample_extractor.sv` - 样本提取器
- **功能**: 从TLWE累加器中提取LWE样本
- **规则**: 
  - `result->a[0] = acc->a[0].coefs[0]`
  - `result->a[j] = -acc->a[0].coefs[N - j]` (j > 0)
  - `result->b = acc->a[1].coefs[0] + mu2`
- **状态**: ✅ 实现完成

#### 4. `bram_simple.sv` - 简单BRAM模块
- **功能**: 提供双端口BRAM接口
- **用途**: 存储累加器、输入LWE等数据
- **状态**: ✅ 实现完成

### 复用模块

#### 共享组件
1. **NTT引擎**: 用于外部积计算
2. **BSK管理器**: 用于BSK数据流管理
3. **AXI接口**: 用于HBM访问
4. **RegFile接口**: 用于输入/输出LWE样本

## 关键设计决策

### 1. 模块化设计
- 将复杂算法分解为独立的功能模块
- 每个模块负责特定的数学操作
- 便于验证和调试

### 2. 资源复用
- 最大化复用现有的NTT引擎
- 共享BSK和AXI接口
- 减少硬件资源消耗

### 3. 状态机设计
- 直接映射C++算法流程
- 清晰的状态转换逻辑
- 便于理解和维护
